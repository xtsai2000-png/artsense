### 2026-03-24 (一) - Artsense 比對頁面修復 + 前處理流程升級

**問題修復：比對頁面卡在「準備中....」**
- **根本原因**：前端 `approveAndSearch()` 中，Stream 最後一個 chunk `done=true` 但 `value` 為空，導致 `decoder.decode()` 未被呼叫，buffer 裡最後一筆 `complete` 事件來不及處理就 break 迴圈
- **修復**：重構 SSE 處理邏輯，`done=true` 時先 `decoder.decode()` flush 殘留資料，再處理 buffer，並加入 `return` 確保 complete 後立即結束
- **驗證**：16 張圖片全部重新處理完成

**Artsense 前處理流程升級：YOLO → SAM → DINOv2**
- 新流程：YOLOv8n 偵測主體 → SAM 精細分割 → DINOv2-base 特徵萃取
- Step 1：YOLOv8n 偵測主體 bounding box
- Step 2a：SAM 以 YOLO bbox 分割（若 YOLO 成功）
- Step 2b：rembg 去背 + SAM 強化（若 YOLO fallback）
- Step 3：DINOv2-base 特徵萃取（768維），存入 ChromaDB
- 16 張圖片全部重新處理完成並入庫

**檔案變更：**
- `main.py`：compare API 從 rembg 改為 YOLO+SAM pipeline（250行）
- `web/templates/compare.html`：SSE buffer 修復 + UI 說明更新
- `src/process_pipeline.py`：新處理腳本（440行）
- `web/static/css/style.css`：小幅修正

**GitHub Push：** commit `68a5223` + `894862e`

### 2026-04-30 (三) - Artsense 審核後台重構

**需求：支援使用者自訂框選藝術品區塊，解決電腦前處理不精確的問題**

**修改檔案：**
- `web/templates/admin.html` — 新增 bbox 框選 modal + 多選支援 + 刪除按鈕
- `src/image_pipeline.py` — 重構為 `segment_artwork()` + `segment_artwork_with_bbox()` + `segment_artwork_with_bboxes()`
- `main.py` — API 重構、覆蓋模式、刪除 API

**功能變更：**

1. **重新處理流程（覆蓋模式）**
   - 每次重新處理直接覆寫同一 `work_id` 的圖檔 + 向量，不長出新卡片
   - 流程：SAM 邊緣切割（支援多 bbox）→ DINOv2 向量萃取 → 覆寫 ChromaDB

2. **Bbox 框選工具**
   - 點「重新處理」→ 跳出 canvas 框選介面（原圖顯示）
   - 拖曳可連續框選多個區塊（Shift 多選）
   - 確認後 SAM 以指定 bbox 重新分割，合併多 mask
   - 框選資料存入 `review_status.json` 的 `manual_bboxes` 欄位

3. **刪除按鈕**
   - 比較視窗左下角有 🗑️ 刪除按鈕
   - 刪除 review_status + 圖檔 + ChromaDB 向量
   - `DELETE /api/admin/delete/{work_id}`

4. **需重處理 tab 移除**
   - 「需重處理」頁面刪除，統一在「待審核」處理
   - 流程簡化為：待審核 → 通過 / 重新處理 / 刪除

5. **卡片圖片 cache-bust**
   - 所有 `/gallery-img/` URL 都加上 `?t=${Date.now()}` 避免瀏覽器快取

6. **approve 最終版本入庫**
   - approve 時 upsert，以 `base_id` 為 id 覆蓋舊向量，確保向量庫只有最終版本

7. **`api/admin/works` 多版本支援**
   - 修正 `return works` duplicate 問題（已修復 unreachable code）
   - 早期版本有 `_2`、`_3` 等多版本卡片（已改覆蓋模式後不再產生新卡片）

- `closeBboxModal is not defined` — 函式未寫入（多次重構過程中漏掉）
